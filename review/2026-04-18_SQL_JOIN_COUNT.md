# 📝 2026年 第1回 情報処理技術者試験 実技 - 誤答ノート (SQL JOIN & COUNT)

## 📌 問題の概要 (13番)
2つのテーブル（`employee`, `dept`）を結合し、条件に合うタプルの数を求めるSQLクエリの実行結果を予測しなさい。

**[テーブルデータ]**
- `employee e` テーブル (社員番号, 社員名, 部署番号)
  - (-, -, **10**)
  - (-, -, **20**)
  - (-, -, **20**)  ← ★ここがポイント
- `dept d` テーブル (部署番号, 部署名, budget)
  - (10, -, 100)
  - (20, -, 300)
  - (30, -, 200)

**[クエリ]**
```sql
SELECT COUNT(*) FROM employee e JOIN dept d
ON e.dept_id = d.dept_id
WHERE d.budget > (select avg(budget) from dept)
```

## ❌ 私の解答 (My Answer)
`1`

## ✅ 正解 (Correct Answer)
`2`

---

## 🔍 なぜ間違えたのか？ (原因分析)

SQLの構文自体は理解していましたが、**データの視覚的な錯覚（思い込み）**と**目視での解法（目コーディング）**が致命的なミスを招きました。

### 1. サブクエリの計算 (正常)
- `(select avg(budget) from dept)`: `(100+300+200) / 3 = 200`
- ここまでは正確に計算し、条件が `WHERE d.budget > 200` であることは把握できていました。

### 2. テーブルデータの錯覚 (根本原因)
- `employee` テーブルの部署番号（10, 20, 20）を、無意識のうちに**「10, 20, 30」という規則的なパターンだと思い込んで（脳内補完して）**しまいました。
- その結果、「両方のテーブルに10, 20, 30が1つずつあるから、JOINしてもそのまま3行になるはずだ」と誤認しました。

### 3. 結合（JOIN）とフィルタリングの誤ったシミュレーション
- 思い込みに基づき、3行の中から `budget > 200` を満たす行を探しました。
  - 部署10 (100) ➔ ❌
  - 部署20 (300) ➔ ✅
  - 部署30 (200) ➔ ❌ (200より大きくないため)
- 結果として、「条件を満たすのは部署20の**1行だけ**だ」と判断し、解答を `1` にしてしまいました。

### 4. 正しい実行プロセス
- `employee` テーブルには部署20のデータが**2行**存在します。
- `dept` テーブル（部署20, budget 300）と結合（JOIN）すると、結果のテーブルには部署20の行が**2行**生成されます。
- `budget > 200` の条件を満たすのはこの2行なので、`COUNT(*)` の結果は **`2`** になります。
