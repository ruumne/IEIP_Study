# 📝 2026年 第1回 情報処理技術者試験 実技 - 誤答ノート (C言語 ポインタ)

## 📌 問題のコード (8番)
```c
#include <stdio.h>

struct fnc {
    int *(*fn)(int*);
};

int *dummy(int *d) {
    return d + 1;
}

int main() {
    int n[] = {16, 32};
    struct fnc mine;
    mine.fn = dummy;
    printf("%x", *mine.fn(n));
    return 0;
}
```

## ❌ 私の解答 (My Answer)
`16`

## ✅ 正解 (Correct Answer)
`20`

---

## 🔍 なぜ間違えたのか？ (原因分析)

今回の問題では、**3つの重要な罠（ポイント）**が隠されていました。

### 1. ポインタ配列の演算 (`d + 1`)
- `dummy`関数内で `return d + 1;` が実行されます。
- ここで `d` は配列 `n` の最初の要素（`16`）のアドレスを指しています。
- ポインタに `+1` をすると、値に1が足されるのではなく、**配列の次の要素（`32`）のアドレス**へ移動します。
- **誤解した点:** ポインタ演算の基本を瞬間的に忘れ、単に最初の値である `16` がそのまま返ってくると勘違いしてしまいました。

### 2. ローカル変数の初期化 (Call by Value)
- 「`dummy`関数の引数 `int *d` はローカル変数だから、初期化されずにゴミ値（Garbage value）が入るのでは？」と疑問に思いました。
- **解決:** 引数もローカル変数ですが、関数が呼び出された瞬間 (`mine.fn(n)`) に、引数として渡された `n` のアドレス値で**自動的に初期化**されるため、ゴミ値ではなく正常なアドレスを持ちます。

### 3. フォーマット指定子 (`%x`)
- 最後の `printf("%x", ...)` で `%x` は**10進数を16進数（小文字）に変換して出力**する指定子です。
- ポインタ演算で得られた値は10進数の `32` です。
- `32` を16進数に変換すると **`20`** になります。
- **誤解した点:** `%x` が16進数変換であることを知らず、自分が導き出した値（`16`）をそのまま10進数で書いてしまいました。

1. **手書きトレーシングの徹底**
   - コードを読むだけ（目コーディング）ではなく、変数とアドレスを図や表で書きながら値の変化を追跡する。
2. **C言語のフォーマット指定子の暗記**
   - `%d` (10進数), `%x` / `%X` (16進数), `%o` (8進数) の違いを確実に覚える。
3. **ポインタ演算のルール再確認**
   - `ポインタ + 1` = 次のメモリアドレスへの移動（データ型サイズ分）であることを常に意識する。
