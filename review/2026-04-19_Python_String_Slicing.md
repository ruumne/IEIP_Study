# 📝 2026年 第1回 情報処理技術者試験 実技 - 誤答ノート (Python 文字列処理)

## 📌 問題のコード (14番)
```python
s = input("入力:") # 入力: HumanDev

x = []
for i in s.split():
    x.append(i[::-1])
y = ' '.join(x)
z = ''.join([c for c in y if c not in 'ong'])
print(z)
```

## ❌ 私の解答 (My Answer)
`HveDamu`

## ✅ 正解 (Correct Answer)
`veDamuH`

---

## 🔍 なぜ間違えたのか？ (原因分析)

Python のスライシングにおける「開始インデックスの省略ルール」に対する誤解が最大の原因でした。

### 1. 文字列のスライシング `[::-1]` の罠
- `[start:stop:step]` の構文で、`step` が `-1`（逆方向）に設定されています。
- 🚫 **誤解した点:** 
  > ~~「開始位置(`start`)が省略されているから、当然 `0番目` のインデックスから始まるはずだ」~~ と論理的に推測してしまいました。
- 💡 **Pythonの実際の挙動:** 
  > `step` にマイナス（負の数）を指定すると、進行方向が逆になります。そのため、開始位置を省略した場合は、自動的に **「文字列の最後尾（-1番目）」** からスタートする仕様になっています。
- したがって、`HumanDev` の一部だけが反転するのではなく、文字列全体が完全に鏡文字のように反転し、`veDnamuH` となります。

### 2. リスト内包表記によるフィルタリング (正常)
- `[c for c in y if c not in 'ong']`
- 反転した文字列 `veDnamuH` から、文字 `c` が `o`, `n`, `g` のいずれかでない場合のみ抽出します。
- ここで `n` が除外されるというロジックは試験中に正確に把握できていました。
- 抽出結果のリスト: `['v', 'e', 'D', 'a', 'm', 'u', 'H']`

### 3. 文字列の結合 (`join`)
- `z = ''.join(...)` で空白なしで結合され、最終出力は `veDamuH` となります。

## 🎯 今後の対策 (Action Plan)
1. **スライシングの「省略の原則」を上書きする**
   - `step` がプラス（`1`など）なら、省略された開始位置は `0`（先頭）。
   - `step` がマイナス（`-1`など）なら、省略された開始位置は **絶対の最後尾** になる！
   - コード問題で `[::-1]` が出たら、「全体を完全にひっくり返す」コマンドだと直感的に判断する。
2. **`split()` の挙動確認**
   - 今回は入力が単一の単語（空白なし）だったため `split()` してもリストの要素は1つ `['HumanDev']` でした。もし入力に空白があれば単語ごとに反転することになるため、入力値の空白の有無も必ずチェックする。
