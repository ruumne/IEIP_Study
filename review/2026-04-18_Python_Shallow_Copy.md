# 📝 2026年 第1回 情報処理技術者試験 実技 - 誤答ノート (Python 浅いコピー)

## 📌 問題のコード (12番)
```python
def f(a):
    m = [[x] for x in a]
    b = m[:]
    for i in range(len(b)-1):
        b[1+i] += b[i]
    result = sum([len(x) for x in m])
    return result

print(f())[1][2][3][4]
```

## ❌ 私の解答 (My Answer)
`4`

## ✅ 正解 (Correct Answer)
`10`

---

## 🔍 なぜ間違えたのか？ (原因分析)

Python のリストにおける **「浅いコピー（Shallow Copy）」と「ミュータブル（可変）オブジェクトの参照」** の性質を完全に見落としていました。

### 1. `m[:]` によるリストのコピーの罠
- `m = [[1], [2], [3], [4]]` という二次元リストが生成されます。
- `b = m[:]` はリストをスライシングして新しいリスト `b` を作成します。
- **誤解した点:** 「`b` をコピーしたから、その後 `b` の中身を変更しても元の `m` には影響しないはずだ」と思い込み、最終的な `m` の長さの合計を `1 + 1 + 1 + 1 = 4` と計算してしまいました。
- **正しい動作:** `m[:]` は**浅いコピー**です。外側のリストは新しく作成されますが、内側のリスト（`[1]`, `[2]` など）は `m` と `b` で**同じメモリアドレスを参照**しています。

### 2. `b[1+i] += b[i]` の影響
- ループ内で `b` の要素（内側のリスト）に対して `+=` 演算が行われます。これはリストの拡張（`extend`）と同じ動作です。
- `b[0]` は `[1]`、`b[1]` は `[2]` であり、`b[1] += b[0]` を実行すると `b[1]` は `[2, 1]` になります。
- **ここがポイント:** 浅いコピーのため、**`m` の要素も同時に変更**され、`m[1]` も `[2, 1]` になります。
- ループが終了した時点での `m` は `[[1], [2, 1], [3, 2, 1], [4, 3, 2, 1]]` となっています。

### 3. 最終結果の計算
- `result = sum([len(x) for x in m])`
- 各要素の長さは `1, 2, 3, 4` となり、その合計は `10` となります。

## 🎯 今後の対策 (Action Plan)
1. **二次元リストのコピーには警戒する**
   - コード問題で `a = b[:]` や `a = list(b)` のようなコピー記述が出た場合、それが一次元リストなら問題ないが、**二次元リスト（リストの中にリストがある場合）なら「浅いコピー」の罠**であることをすぐに疑う。
2. **参照（Reference）の概念を視覚化**
   - 手書きトレーシングの際、変数のコピーを別の箱として描くのではなく、「同じ箱を指す矢印」として描き、一方が変われば他方も変わることを物理的に追跡する。
